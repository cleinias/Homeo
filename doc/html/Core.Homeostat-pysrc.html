15  <<<<<<< Updated upstream 
16      Homeostat manages the complete homeostat by taking care of the communication between the units and between the Units and the Uniselector. 
17      It stores a collection of units, and some state variables representing the general state of the Homeostat at any point in time.  
18      "Starting" (an instance of) this class is equivalent to turning the switch on the electro-mechanical machine built by Ashby.  
19      Notice that this is typically done by the application class HomeoSimulation, which offers facilities for adding units,  
20      adding connections, selecting parameters pertaining to the homeostat (weights, etcetera) and pertaining to the simulation  
21      (number of iterations, print out and/or display of data, etcetera). Homeostat collaborates with DataCollector  
22      (an instance of which it holds) to record its states for any instant of time it goes through. However, it does not contain any facility 
23       for visualizing the data themselves.  If operated manually, an instance of Homeostat requires manually setting up the various  
24      parameters and does not offer any output. 
25       
26      Instance Variables: 
27          homeoUnits           <Collection>     the collection of homeoUnits making up the homeostat 
28          microTime            <aNumber>        the temporal scale regulating the communication among units (typically identical to the unit time) 
29          time                 <aNumber>        the current time index (i.e., t) 
30          dataCollector        <aDataCollector> the object recording the states of the homeostat 
31          collectsData         <aBoolean>       whether or not the homeostat collects data about its run 
32          slowingFactor:       <milliseconds>   it slows down the simulation by inserting a slowingFactor wait after each cycle. 
33          isRunning            <aBoolean>       whether the homeostat is running

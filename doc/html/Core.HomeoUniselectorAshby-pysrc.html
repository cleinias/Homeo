 10      ''' 
 11      HomeoUniselectorAshby is a different implementation of Ashby's (sparsely described!) algorithm. 
 12      It simulates a stepping switch (or Uniselector) that advances one step every time it is activated. 
 13      At every step the uniselector produces weight values for the HomeoUnits connections.  
 14      The simulation is implemented  with a m x n matrix (the ivar matrix) where m  
 15      are the steps of the uniselector, and n are the number of connected units.  
 16      In Ashby's classical homeostat m= 25, n = 3.  The values in the matrix are produced according  
 17      to three different algorithms, each a different intepretation of Ashby's original idea.  
 18      The parameter deciding which production method to use is held in ashbyKind. 
 19   
 20      Instance Variables: 
 21      unitsControlled <anInteger>       The maximum number of units this uniselector controls. Default values is 10. It will be incremented in 10-units increment if needed 
 22      unitIndex       <anInteger>        The index keeping track of which unit's connection we are currently producing a value (a weight) for. 
 23      ashbyKind       <aString>          The string determining which initialization (and re-initialization) method to use. 
 24      unitIndex 
 25      interval 
 26      steps        <anInteger>       The number of possible steps in the Uniselector (default is 12, as per Ashby's implementation) 
 27      index        <anInteger>       The index keeping track of which row of weight we should output next

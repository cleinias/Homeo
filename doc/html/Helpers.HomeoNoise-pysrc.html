 11      ''' 
 12      HomeoNoise is a utility class that provides different algorithms to deal with  
 13      the computation of how noise affects the transmission of current between units  
 14      and the flicker noise that may affect the units' internal values.  
 15   
 16      Instance variables: 
 17   
 18      current       <aNumber>    The value of the current affected by the noise 
 19      noise         <aNumber>    The value of the noise affecting the transmission of the current. This value is typically between 0 (no noise) and 1 (channel so noisy to block communication) 
 20      mode          <aString>    Mode of the noise. Possible values: 'distorting' 'degrading', indicates how the noise affects the currentWhether the noise is a         
 21      distribution  <aString>    Probability distribution of the actual noise with respect to the noise value. Possible values: 'constant', 'uniform', 'normal' 
 22      ratio         <aString>    Ratio of the noise to the affected current. Possible values: 'linear' 'proportional' 
 23   
 24      It is a singleton class.  
 25      The only class creation method is newWithCurrent: andNoise: which simply updates the values of  
 26      the instance variables if the unique instance exists already. 
 27   
 28      The noise-computing algorithm is selected by changing the values of  
 29      the three iVars mode, distribution, and ratio, and then calling the method getNoise.  
 30      This method constructs a selector string from the values of the ivar and calls the corresponding method.  
 31
